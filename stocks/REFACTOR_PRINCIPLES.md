# REFACTOR_PRINCIPLES.md - 本轮重构的核心纠偏

## 这次纠偏在纠什么

本轮重构明确纠偏：

- 不再把系统做成“规则先写结论，LLM 后润色”
- 不再把用户资产过度压缩成几个固定标签
- 不再让程序的中间层输出替代最终分析
- 不再让系统只盯 `personal-latest.json` 单帧快照

新的正式方向是：

**这个系统主要靠 LLM 发挥和驱动，程序负责提供完整上下文、结构化脚手架、历史 snapshot 和记工程健壮性。**

---

## 为什么这件事重要

如果系统只是：

- 收集资产
- 收集新闻
- 拼几条规则
- 再把内容发出去

那它本质上只是一个自动化信息转发器。

而这个项目真正有价值的地方是：

- LLM 能理解用户组合结构
- LLM 能识别用户偏好与市场节奏之间的张力
- LLM 能跨资产、跨市场、跨时间做综合判断
- LLM 能结合最近几版变化理解“这不是一天的噪音，还是一段持续状态”
- LLM 能输出真正个性化的解释，而不是标签复读

---

## 重构后的正式原则

1. LLM 是主分析引擎
2. 结构化层是脚手架，不是最终答案
3. 宽输入，窄输出
4. 健壮性必须保留，但不能以牺牲 LLM 分析空间为代价
5. 用户资产分析优先于纯行情转述
6. 历史 snapshot 必须可持续累积，供 LLM 理解阶段性变化

---

## 对后续开发的约束

后续无论新增功能、重写 prompt、补测试还是改文档，都必须检查：

- 是否给了 LLM 足够完整的上下文
- 是否把中间信号误写成了最终结论
- 是否把用户资产结构压缩得过狠
- 是否让规则系统偷走了本该属于 LLM 的分析空间
- 是否仍然保留了日志、快照、失败兜底等工程完整性
- 是否让 LLM 看到最近几版变化，而不只是 latest 单版状态

只要答案偏向“是”，就说明系统又在走回头路。

---

## 一句总纲

**这个系统不是为了更整齐地转发信息，而是为了让 LLM 在稳固工程地基和连续历史上下文上，真正分析人的金融结构。**

---

## 新增收口约束（2026-03-28）

当前开始执行收口：

- 不再继续横向扩张文档体系
- 不再新增“未来也许有用”的独立项目支线
- 不再默认新增更多中间状态层
- 若一个新模块不能明显提升当前用户价值（资产维护、建议质量、投递稳定性、关键测试），就不该进入主线

后续开发默认先问一句：

> 这是真的让当前 `stocks` 更有用，还是只是让架构看起来更完整？

如果是后者，就不做
